The problem

Capable. Unconstrained.

AI agents write code fast — and drift just as fast. Three failure modes, every run.

They over-build

Scope creep past the spec — features, files, and abstractions nobody asked for.

They skip verification

Phases close without ever proving the exit criteria were actually met.

They leave no record

No audit trail, no rollback point. When something breaks, you can't see why or undo it.

The keel loop

Four guarantees, every phase.

Run keel init and it installs a harness of commands, rules, and gate files the agent operates inside — markdown it reads before it writes.

01

Rules before code

The agent reads agent-rules.md — your repo constitution — before it writes a single line. Scope, tech stack, and conventions, declared up front.

02

One phase at a time

Each /run-phase N runs exactly one phase, governed by its phase TDD. The gate file doesn't get written until the exit criteria actually pass. No skipping ahead.

03

Audit on every action

Every phase, tool call, and token is appended to an events.jsonl ledger. Query it after the fact with keel report.

04

Reversible by design

Each phase writes a rollback DAG. keel rollback N unwinds a phase and everything downstream — back to a known-good keel.
